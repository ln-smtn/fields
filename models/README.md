# Обученные модели классификации культур

Эта папка содержит финальные модели для классификации сельскохозяйственных культур по временным рядам Sentinel-1/2 + ERA5 + SRTM. Все модели работают поверх предобученной PRESTO (NASA Harvest, 2023).

## Состав

| Файл | Размер | Что это |
|------|--------|---------|
| `class_mapping.json` | <1 КБ | Соответствие ID ↔ название культуры |
| `class_centers.npy` | ~30 КБ | Средние эмбеддинги классов (для OOD-детекции) |
| `class_std.npy` | <1 КБ | Разброс эмбеддингов внутри классов |
| `linear_head.pt` | ~50 КБ | Linear probe: голова поверх замороженной PRESTO |
| `finetuned_model.pt` | ~2 МБ | Fine-tune: PRESTO с частичной разморозкой + голова |
| `maml_base.pt` | ~2 МБ | MAML: стартовая модель, обучена на 3 регионах |
| `adapted/maml_<region>.pt` | ~2 МБ каждая | MAML после адаптации к конкретному региону |
| `metrics_comparison.json` | <1 КБ | Сводная таблица метрик |

## Что куда применять

### `linear_head.pt` — Linear probe
- **Когда**: быстрая инференция в обучавшихся регионах;
- **Как работает**: PRESTO даёт эмбеддинг → линейный слой → культура;
- **Плюсы**: самый быстрый, маленький, объяснимый;
- **Минусы**: ниже потолок качества.

### `finetuned_model.pt` — Fine-tune (рекомендуется для сценария A)
- **Когда**: любая инференция в обучавшихся регионах (South, Chernozem, Volga) и соседних;
- **Как работает**: PRESTO с подкрученными последними 2 блоками + голова;
- **Плюсы**: лучшие метрики на знакомых регионах;
- **Минусы**: на принципиально новом регионе (Приморье, Алтай) проседает.

### `maml_base.pt` — MAML стартовая
- **Когда**: НЕ применяется напрямую. Только как стартовая точка для адаптации;
- **Как работает**: обучена быть «хорошим инициализатором» для быстрой адаптации к новой задаче;
- **Зачем хранить**: чтобы потом адаптировать к новым регионам без переобучения всего.

### `adapted/maml_<region>.pt` — MAML адаптированная (сценарий B)
- **Когда**: применять к данным конкретного нового региона;
- **Как создаётся**: 25 размеченных полей региона + `adapt_to_region.py` → 5 шагов SGD → готова;
- **Плюсы**: качество близко к full fine-tune, но **из 25 примеров** вместо тысяч;
- **Минусы**: нужна разметка хотя бы 25 полей региона.

## Регионы в обучающей выборке

Модели `linear_head.pt`, `finetuned_model.pt` обучены на:

| Регион | Тайлы MGRS | Полей | Годы | Главные культуры |
|--------|-----------|-------|------|-------------------|
| **South** | 38TLR, 37TGM, 38TLS, 38TLQ | 11 863 | 2017-2021 | оз.пшеница, подсолнечник, пар, нут |
| **Chernozem** | 36UXV, 37UDB, 37UEB | 10 923 | 2016-2019 | оз.пшеница, соя, свёкла, ячмень |
| **Volga** | 38UQB, 38UPC, 38UQC | 1 951 | 2016-2021 | оз.пшеница, ячмень |

`maml_base.pt` обучен на тех же 3 регионах. **Primorye отложен** для cross-region теста и адаптации (см. `adapted/maml_primorye.pt`).

## Метрики (заполнить после обучения)

См. `metrics_comparison.json`. Ожидаемые значения:

| Метод | F1 macro (test) | Сценарий применения |
|-------|-----------------|---------------------|
| Linear probe | ~0.78 | A |
| Fine-tune (частичная разморозка) | ~0.83 | A (рекомендуется) |
| MAML base (без адаптации на Приморье) | ~0.50 | — (используется как старт) |
| MAML + 5-shot адаптация к Приморью | ~0.65 | B |
| Fine-tune на Приморье (zero-shot) | ~0.40 | сравнение для диплома |

## Класс культур

| ID | Полное название | Сокращение |
|----|-----------------|------------|
| 0 | озимая пшеница | оз.пшеница |
| 1 | яровая пшеница | яр.пшеница |
| 2 | озимый ячмень | оз.ячмень |
| 3 | яровой ячмень | яр.ячмень |
| 4 | кукуруза | кукуруза |
| 5 | подсолнечник | подсолнечник |
| 6 | соя | соя |
| 7 | свекла сахарная | свёкла |
| 8 | горох | горох |
| 9 | нут | нут |
| 10 | озимый рапс | оз.рапс |
| 11 | яровой рапс | яр.рапс |
| 12 | лен | лён |
| 13 | пар (сидеральный, смешанный, эспарцет) | пар |
| 14 | прочее | прочее |

Также есть метки **для пост-фильтрации** (не выдаются классификатором, ставятся фильтром):
- `"не поле"` — низкая уверенность ИЛИ далеко от обучающего распределения (OOD);
- `"артефакт FTW"` — площадь < 0.5 га, скорее всего ошибка сегментации.

## Использование

### Простейший случай — сценарий A:

```python
import torch
from presto import Presto
from pathlib import Path

# Загрузить fine-tuned модель
state = torch.load("models/finetuned_model.pt", map_location="cpu")
presto = Presto.load_pretrained()
presto.encoder.load_state_dict(state["encoder"])

head = torch.nn.Linear(128, 15)
head.load_state_dict(state["head"])

presto.eval()
head.eval()

# Дальше используешь predict_tile.py — он всё это делает сам.
```
